Game Overview

City Slicker is a fast-paced movement-based first-person shooter developed completely solo in Unity using C#. The game was created over five weeks with the goal of experimenting with responsive and high-speed traversal.

All gameplay systems, mechanics, and assets were built from scratch without external packages or libraries.

Development

City Slicker focuses on building up speed to jump from building to building while eliminating turrets and avoiding traps.

The goal was to create a movement system that feels fluid, fast, and satisfying to master while still rewarding risk-taking.

During development I ran into several limitations with Unity’s Animator system. I eventually replaced it entirely with code-driven animation triggers, giving me more control and reliability.

The project also helped deepen my understanding of object-oriented programming concepts such as polymorphism, encapsulation, and inheritance.

What I Learned

  • Improved understanding of core OOP concepts
  • Learned the strengths and limitations of Unity’s animation system
  • Developed modular and reusable code structures
  • Gained experience building a complete project solo
