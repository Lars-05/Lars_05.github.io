Project Overview


For this project I had to script a dungeon generator that worked with any width or height, it also had to come with a player that could navigate through the dungeon using a pathfinding algorithm. The dungeon generator has the following features:

• The dungeon will never loop

• Assets are placed using marching squares and flood fill, allowing corner pieces

• Assets can be easily swapped out

• Player movement supports A*, Dijkstra, and NavMesh, with visual comparisons between algorithms

• The entire dungeon generation process can be visualized with gizmos, making debugging easier

What Did I learn?


• What algorithms are and how to describe them using pseudocode and flowcharts

• Advanced debugging techniques beyond using Debug.Log

• Understanding HashSets, Queues, and Dictionaries and when to use them

• A* and Dijkstra: how they work and how to implement them

• BFS, DFS, Flood Fill, and Marching Squares: how they work and how to implement them

• What Big O notation is and how to apply it to an algorithm

• How to implement algorithms both iteratively and recursively
